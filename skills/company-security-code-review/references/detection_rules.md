# 风险分级与证据标准

## 风险等级

- `critical`
  - 可直接导致远程命令执行、严重数据泄露、认证绕过、任意文件访问
- `high`
  - 高概率导致越权、敏感信息泄露、重要控制缺失
- `medium`
  - 有明确风险，但需要额外前提或影响面较窄
- `low`
  - 最佳实践缺失、调试残留、治理类问题

## 判定要求

- 只有在看到真实代码证据时才写 `不符合`
- 如果只能推测、需要依赖部署态或外部系统，则写 `需人工确认`
- 如果一个问题可以构成明确攻击链，即使只有单点证据，也优先上报

## 证据最低标准

每个发现至少给出：

- 文件路径
- 行号
- 原始代码片段
- 对应公司条款
- 为什么构成风险
- 为什么现有控制不足

## 常见降误报方式

- 查同一调用链路上是否已做服务端校验
- 查是否使用了白名单、编码或参数化查询
- 查是否在中间件、过滤器、装饰器或依赖注入层补了鉴权
- 查是否只是测试代码、示例代码或注释
