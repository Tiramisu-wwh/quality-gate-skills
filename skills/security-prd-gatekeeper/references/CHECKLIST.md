# 安全设计评审参考

本文件用于帮助 agent 在需求和设计阶段做安全左移评审。

## 1. 结果值规则

每一项只允许使用以下结果值：

- `满足`
- `不满足`
- `部分满足`
- `未涉及`

判定建议：

- `满足`：文档里已经明确写出控制措施、触发条件、异常策略或验收口径。
- `部分满足`：方向对了，但缺少边界、限制、失败处理、日志、权限或验收条件。
- `不满足`：该能力对当前需求适用，但文档里没写或明显缺失。
- `未涉及`：业务范围确实没有该类能力，例如没有文件上传、没有短信验证码、没有密码找回。

## 2. 评审对象与阶段定位

本技能关注的是“文档是否明确安全设计要求”，不是检查代码是否已经实现。

适用输入：

- PRD / BRD
- 技术方案
- 接口文档
- Figma 设计稿中的页面交互与字段说明

文档阶段常见判断方式：

- PRD 中应关注：敏感信息、角色权限、验收标准、异常提示、日志与审计要求。
- 技术方案中应关注：鉴权方案、密钥管理、加密算法、会话策略、运行环境、文件上传下载控制。
- 接口文档中应关注：敏感字段、传输方式、鉴权、参数校验、错误返回、接口调用安全。

## 3. 16 个安全域

### 3.1 敏感信息

关注：

- 响应字段最小化
- 脱敏展示
- 敏感信息不经 GET 传输
- 传输与存储加密
- 禁止明文硬编码与明文保存

### 3.2 输入校验

关注：

- 服务端最终校验
- 类型/长度/范围校验
- 危险字符过滤或转义
- URL 重定向白名单

### 3.3 输出编码

关注：

- HTML/URL/XML/LDAP 等场景编码
- 不可信数据输出安全处理

### 3.4 异常处理

关注：

- 不暴露堆栈、数据库结构、源码、敏感字段
- 失败回滚
- 资源释放
- 精确捕获异常

### 3.5 身份认证

关注：

- 非公开资源必须认证
- 登录失败锁定
- 失败提示模糊化
- MFA / 二次验证
- 外部接口身份验证

### 3.6 会话安全

关注：

- 会话随机性
- 注销与会话终止
- 超时策略
- Cookie `HttpOnly` / `Secure`
- 防会话固定
- token 安全性

### 3.7 访问控制 / 权限管理

关注：

- 最小权限原则
- 资源级权限校验
- 外部应用访问控制
- 服务器端鉴权
- 端口开放最小化

### 3.8 密码管理

关注：

- 密码复杂度
- 首次登录修改
- 密码存储算法
- 密码不写日志
- 临时密码有效期

### 3.9 密码找回安全

关注：

- 服务端校验
- 二次验证
- 多步验证顺序
- 手机号与验证码绑定校验

### 3.10 短信验证码

关注：

- 一次一用
- 频率限制
- 有效期
- 用户名/手机号/验证码三元绑定

### 3.11 图形验证码

关注：

- 一次一用
- 安全随机生成
- 扭曲与干扰
- 服务端最终校验

### 3.12 日志规范

关注：

- 敏感信息不落日志
- 日志访问受控
- 用户关键操作留痕
- 六要素完整

### 3.13 文件上传下载

关注：

- 身份验证
- 类型白名单
- 服务端校验
- 禁止可执行文件
- Zip 解析安全
- 访问路径安全

### 3.14 接口安全

关注：

- 接口间身份认证
- 参数不可遍历
- 资源 ID 安全设计

### 3.15 加密算法及密钥管理

关注：

- 使用公开安全算法
- 密钥单一用途
- KMS 或分层密钥管理
- TLS 1.2+
- 安全随机数

### 3.16 运行环境安全

关注：

- 生产环境去除调试与测试代码、配置、文件

## 4. 文档阶段的高优先级必查项

如果当前需求涉及以下能力，这些项原则上不能空缺：

- 登录/注册/密码：`5.*`、`6.*`、`8.*`
- 文件上传下载：`13.*`
- 外部接口或系统集成：`14.*`
- 敏感个人信息：`1.*`、`12.*`、`15.*`
- 角色和数据隔离：`7.*`

## 5. 结论建议

可使用以下口径给出结论：

- `通过`：高风险项已明确，无关键安全域空白。
- `有条件通过`：已有设计方向，但存在若干需补齐项，修订后可进入下一阶段。
- `不通过`：关键安全域缺失明显，继续开发会把安全问题后移到实现阶段。
