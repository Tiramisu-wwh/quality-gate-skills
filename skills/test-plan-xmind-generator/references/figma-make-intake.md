# Figma Make Intake

Use this reference when the user includes a Figma Make artifact together with requirement inputs.

## What To Extract

- Page list and navigation order
- Modal, drawer, and tab states
- Field labels, action labels, and visible status names
- Main user flows and alternative branches
- Obvious backend touchpoints such as APIs, service calls, or mock data structures

## Priority

- PRD defines business purpose and acceptance intent
- Figma Make defines visible flow, wording, and state transitions
- Technical solution and API docs define implementation constraints

## If Inputs Conflict

- Record the conflict in assumptions or risks
- Do not silently merge contradictory rules
- Keep the final test plan and cause map aligned to the stated assumption
