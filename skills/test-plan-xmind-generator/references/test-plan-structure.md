# 测试计划结构

Markdown 输出应保持紧凑，并以决策和执行为导向。

## 必填章节

1. `概述`
2. `测试范围`
3. `测试策略`
4. `重点模块与测试关注点`
5. `风险与依赖`
6. `测试环境`
7. `假设`

## 编写要求

- 测试范围应清楚区分范围内与范围外事项。
- 测试策略应强调覆盖意图，而不是执行细枝末节。
- 模块章节应同时体现需求拆分和测试关注点。
- 风险应优先写会影响排期优先级或形成阻塞依赖的问题。
- 默认使用通用中文测试语言，不把项目内部字段名、表名、接口路径直接当作正文主语。
- 只有在影响测试判定、联调定位或与接口文档核对时，才保留项目专有术语。
- 若必须保留专有术语，优先采用“中文说明（专有名词）”的写法。
